# Hoodlums Country Club — Print-Ready Design Prompts

> 14 prompts for **flat artwork** (the file Printful prints onto blank apparel) — not mockups.
> Brand: **Hoodlums Country Club** | Tagline: *"Bad Decisions · Good Times."*

## How to use this file

1. Paste each prompt into ChatGPT (DALL-E 3) or your image model of choice.
2. **Save the output as a PNG with a transparent background**, using the filename listed under each prompt.
3. Drop the file into `/Users/williambeltz/Documents/countrie-hoodlums/artwork/`.
4. Tell Claude "done" — it will create Printful sync products via the API and wire each variant ID into `mock-data.ts`.

## Universal style direction (apply to every prompt)

> **Flat 2D vector-style illustration on a fully transparent background. No garment, no shirt, no hoodie, no model, no mannequin, no photo background. Centered composition with margin around the edges. Square 1:1 canvas, minimum 4500 x 4500 pixels at 300 DPI. High color saturation and strong contrast so the design reads cleanly when printed on dark fabric. No drop shadow from objects. No simulated fabric texture. Output as a single isolated graphic ready for screenprint or DTG production.**

If the model insists on adding a background, append: *"isolated on pure white background, ready for vector tracing"* — easier to clean up than a colored background.

## After generation

Once all 14 PNGs are in `/artwork/`, Claude will:
1. Upload each design file to Printful via `/files` endpoint.
2. Look up matching Printful catalog blanks (e.g. Heritage Hoodie → Gildan 18500 black hoodie).
3. Create sync products via `/store/products` with the design attached to each blank size variant.
4. Write the resulting `sync_variant_id` per size into [src/lib/mock-data.ts](../src/lib/mock-data.ts) under each product's `printfulVariantIds`.
5. Surface the first draft order through `/admin/orders` after a Stripe checkout test.
